# Юридическая информация

Эта страница фиксирует правовой статус материалов, которые лежат в репозитории или используются при сборке документации и PDF. Она не заменяет текст лицензий и не является юридической консультацией.

## Лицензия проекта

В корне репозитория находится файл `LICENSE.md`. В нем используется смешанная схема лицензирования:

| Материалы | Лицензия |

| Код, скрипты, Docker/Task-конфигурация и вспомогательная автоматизация | PolyForm Noncommercial License 1.0.0 |
| Документация, авторский текст, LaTeX-исходники, диаграммы и поясняющие материалы | Creative Commons Attribution-NonCommercial 4.0 International |
| Шрифты в `fonts/latin-modern-mono/` | GUST Font License из `fonts/latin-modern-mono/GUST-FONT-LICENSE.TXT` |

Некоммерческое использование разрешается на условиях соответствующих лицензий. Коммерческое использование не предоставляется автоматически и требует отдельного письменного согласования с автором.

К коммерческому использованию могут относиться, например, включение материалов в платный продукт, платный курс, коммерческую консультацию, закрытую корпоративную документацию или сервис.

## Шрифты

В каталоге `fonts/latin-modern-mono/` лежат файлы семейства Latin Modern Mono:

- `*.otf` - файлы шрифтов;
- `GUST-FONT-LICENSE.TXT` - текст лицензии;
- `lm-info.pdf` - справочная информация о семействе Latin Modern.

Согласно `fonts/latin-modern-mono/GUST-FONT-LICENSE.TXT`, эти шрифты распространяются по GUST Font License версии 1.0 от 22 июня 2009 года. Лицензия разрешает распространение и изменение работы на условиях LaTeX Project Public License версии 1.3c или любой более поздней версии.

При создании производных версий шрифтов лицензия просит, хотя юридически не требует, менять названия шрифтов и файлов, входящих в производную работу. Полный и обязательный текст условий находится в самом файле лицензии: `fonts/latin-modern-mono/GUST-FONT-LICENSE.TXT`.

Файл `LICENSE.md` не меняет лицензию этих шрифтов и не выдает дополнительных прав на них.

## Сторонние инструменты

Репозиторий содержит конфигурацию и инструкции для сторонних инструментов, но сами эти инструменты поставляются по собственным лицензиям:

| Инструмент | Где используется |
| --- | --- |
| Docker и Docker Compose | воспроизводимая сборка профилей |
| TeX Live, LuaLaTeX, latexmk, biber, Python | сборка LaTeX-документа |
| Zensical | сборка документации |
| Mermaid CLI | генерация Mermaid-диаграмм |
| Python-библиотеки из `requirements.txt` | вспомогательные скрипты и диаграммы |
| diff-pdf | визуальное сравнение PDF |

Перед распространением собранных артефактов или Docker-образов нужно учитывать лицензии этих зависимостей и образов.

## Изображения и документы

Изображения в `docs/ru/assets/`, `.github/images/`, `figures/` и PDF/DOCX-файлы могут содержать производные материалы: скриншоты интерфейсов, экспортированные диаграммы, титульные страницы, задание и фрагменты итогового диплома. Их правовой статус может отличаться от статуса исходного кода.

Перед публичной публикацией рекомендуется проверить:

- не содержат ли скриншоты персональные данные или служебную информацию;
- можно ли распространять титульный лист и задание по правилам учебной организации;
- не попали ли в PDF локальные пути, имена пользователей или приватные данные;
- соответствует ли публикация требованиям к дипломной работе.

## Локальные пути и приватные данные

Файл `.env` не коммитится и предназначен для локальных путей, например `VAULT_OS_PATH`. Не добавляйте в репозиторий приватные ключи, токены, абсолютные пути к личным каталогам и закрытый исходный код, если на него нет разрешения.
